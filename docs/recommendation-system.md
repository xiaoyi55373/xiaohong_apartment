# 小红公寓claw - 智能推荐系统

## 📋 概述

智能推荐系统是"小红公寓claw"项目的核心功能之一，通过分析用户的浏览历史和行为数据，为用户提供个性化的房源推荐。

## 🎯 功能特性

### 1. 混合推荐策略
系统采用三种推荐算法的混合策略：
- **基于内容的推荐 (50%)**：根据用户历史浏览偏好推荐相似房源
- **协同过滤推荐 (30%)**：根据相似用户的行为推荐房源
- **热门推荐 (20%)**：推荐当前热门的房源

### 2. 推荐类型

| 推荐类型 | 说明 | 适用场景 |
|---------|------|---------|
| CONTENT | 基于内容的推荐 | 有浏览历史的用户 |
| COLLABORATIVE | 协同过滤推荐 | 有相似用户的场景 |
| HOT | 热门推荐 | 新用户或冷启动 |

### 3. 用户偏好分析
系统自动分析用户的以下偏好：
- 偏好区域（基于浏览历史中的区域分布）
- 价格范围（基于浏览房源的租金统计）
- 标签偏好（基于浏览房源的标签权重）
- 浏览频次和时间分布

## 🔧 算法详解

### 1. 基于内容的推荐算法

基于用户历史浏览记录，分析用户的偏好特征，计算候选房源与偏好的匹配度。

**得分计算公式：**
```
总得分 = 区域匹配得分 × 0.3 + 
         价格匹配得分 × 0.25 + 
         标签匹配得分 × 0.25 + 
         新鲜度得分 × 0.1 + 
         热度得分 × 0.1
```

**区域匹配得分：**
- 根据用户浏览历史中各区域的权重计算
- 权重 = 该区域浏览次数 / 总浏览次数

**价格匹配得分：**
- 计算候选房源租金与用户平均租金的差异百分比
- 得分 = 1 - |租金差异| / 用户平均租金

**标签匹配得分：**
- 统计用户浏览房源的标签分布
- 匹配度 = 候选房源标签权重之和

### 2. 协同过滤推荐算法

基于用户相似度的推荐算法，找到与当前用户兴趣相似的其他用户，推荐他们浏览过的房源。

**用户相似度计算（Jaccard相似度）：**
```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```
其中：
- A：用户A浏览的房源集合
- B：用户B浏览的房源集合
- |A ∩ B|：共同浏览的房源数
- |A ∪ B|：浏览房源的并集

**推荐得分：**
```
推荐得分 = 相似度 × 时间权重
时间权重 = exp(-天数 / 7)  // 7天减半
```

### 3. 热门推荐算法

基于浏览次数统计的热门房源推荐。

**热度计算：**
```
热度得分 = min(浏览次数 / 100, 1.0)
```

优先推荐近30天内浏览次数最多的房源。

## 📡 API接口

### 1. 获取个性化推荐
```
GET /app/recommend/rooms
```

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| limit | Integer | 否 | 推荐数量，默认10 |
| includeViewed | Boolean | 否 | 是否包含已浏览，默认false |
| type | String | 否 | 推荐类型：ALL/CONTENT/COLLABORATIVE/HOT |

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "roomNumber": "101",
      "rent": 3000,
      "recommendScore": 0.85,
      "recommendType": "CONTENT",
      "recommendReason": "符合您的预算，在您常看的朝阳区"
    }
  ]
}
```

### 2. 获取相似房源
```
GET /app/recommend/similar/{roomId}
```

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| roomId | Long | 是 | 房间ID |
| limit | Integer | 否 | 推荐数量，默认6 |

### 3. 获取用户偏好
```
GET /app/recommend/preference
```

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1,
    "browseCount": 15,
    "avgRent": 3500,
    "minRent": 2500,
    "maxRent": 4500,
    "preferredDistrictIds": [1, 2, 3],
    "preferredLabels": ["精装修", "近地铁"],
    "districtWeights": {"1": 0.5, "2": 0.3, "3": 0.2},
    "labelWeights": {"精装修": 0.4, "近地铁": 0.3}
  }
}
```

### 4. 刷新推荐缓存
```
POST /app/recommend/refresh
```

## 💾 缓存策略

### 缓存键设计

| 缓存类型 | Key | 过期时间 |
|---------|-----|---------|
| 用户推荐 | `xiaohong:recommend:user:{userId}:{type}:{limit}` | 30分钟 |
| 用户偏好 | `xiaohong:recommend:pref:{userId}` | 1小时 |
| 热门推荐 | `xiaohong:recommend:hot:{limit}` | 20分钟 |
| 相似房源 | `xiaohong:recommend:similar:{roomId}:{limit}` | 30分钟 |

### 缓存更新策略

1. **定时更新**：热门推荐每20分钟自动刷新
2. **手动刷新**：用户可调用刷新接口更新推荐
3. **事件触发**：用户浏览新房源后，异步更新推荐

## 📊 性能优化

### 1. 数据库优化
- 为browsing_history表的user_id和room_id字段创建索引
- 使用覆盖索引优化热门房源查询

### 2. 算法优化
- 限制相似用户数量为20个
- 限制候选房源数量为前100个最新房源
- 使用缓存减少重复计算

### 3. 异步处理
- 用户行为日志异步记录
- 推荐结果异步缓存
